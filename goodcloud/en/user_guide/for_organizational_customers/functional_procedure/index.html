    Functional Procedure¶

    1. Process Overview¶

    2. Join the Organization¶

    • If a user has not yet registered for a GoodCloud account, an email invitation will be sent to their address when adding them as an organization administrator. Once the user clicks the registration link, they will be automatically enrolled in the organization.

      • Once the user clicks on the link to register, they will be automatically enrolled in the organization.

      • To sign up for a GoodCloud account: Visit GoodCloud, sign up then sign in. If you don't find the verify email, look in spam or check email later. If you have any difficulty with sign up, please send email to b2bsupport@gl-inet.com for help.

    • If the user has already registered for a GoodCloud account, there are two methods to confirm the invitation:

      • Method 1: An invitation email will be sent. By clicking the link and authorizing it, the user will join the organization, and the page will redirect to the organization's view.

      • Method 2: Upon logging into GoodCloud, a reminder will pop up. Clicking 'Accept' will join the user to the organization and redirect them to the organization's view.

    • Users who have joined the organization can view their status in the upper right corner of the interface and switch between the organization view and the SELF-view. Devices and data are not shared between individual and organizational views.

    Note: On the home page, you can view the bound online devices, offline devices, inactive devices, total number of devices, and departmental status, and click the corresponding view module to realize the quick jump of the data source.

    3. Creation of departments within the organization¶

    In the Organization view, navigate to the Administration → Departments page.

    • The page displays the hierarchical structure of each department and provides information about each department (department name, members, and number of devices).

    Note: Members within a department can only view devices in their own department and its sub-departments.

    • When the page opens, the primary department (the organization) is automatically selected. Click '+ Add' to create a secondary department.

    • In the pop-up window, enter the department name and click 'Confirm' to create the secondary department.

    • To add different levels of departments, re-select the department you want to add a sub-level to from the list on the left and repeat the previous steps. Note that if the organization requires a more detailed division of authority, it can be divided into up to five levels of departments.

    4. Create or Modify Organizational Roles¶

    To access the role view status, navigate to Administration → Roles page in the Organization view. The interface will display the current status of the role view.

    • On the left side of the page, you will see a list of roles available for the organization. The organization is created with preset roles, and administrators or members with appropriate permissions can also create new roles as needed. On the right side of the page, the interface displays the actions that can be performed by the currently selected role.

    • To add a new role, click the 'Add a New Role' button in the list. To modify an existing role, select the role and click the 'Edit' button in the bottom right corner of the screen. This will take you to the role editing page where you can make the desired changes.

      ① On the role editing page, you can change the role name by clicking on the field at the top right of the page and entering the new name.

      ② In the right area of the role editing page, you can view a list of all the functional modules available for the organization. You can control the corresponding operation privileges of each role by toggling the buttons beside each module. It is essential to note that all functional modules must have View privileges before you can control other operational privileges for the corresponding module.

    • In addition to toggling individual module privileges, users can also click on "Allow All" or "Close All" in the upper right corner to batch allow or prohibit the corresponding module privileges.

    • Once you have finished configuring the role and made all the desired changes, click the "Apply" button. This will save the changes and return you to the role configuration page.

    5. Inviting Members to Join the Department¶

    To access the Member Management page in the Organization view, go to the Administration → Members page.

    • On the left side of the Member Management page, you will see the department tree. On the right side of the page, you will find the action buttons and the list of members in each department. To invite new members, click the '+ Invite' button and follow the two-step process to complete the invitation.

      • Step 1: enter the email address or username of the user you want to invite in the pop-up window. You can also specify the expiration time for the invitation. Once you have entered the relevant details, click the 'Next' button to proceed to the next step.

      • Depending on the user information entered in the previous step, the system performs different actions:

        • If the email address entered is not yet registered, the interface will pop up a window asking whether to invite the user to register. Click the confirmation to enter the second step, and the system will automatically send an email to the user. After receiving the email, the user will click the link in the email to enter the registration page and will be able to join the corresponding department after successful registration.

        • If the entered email address or user name is already registered, the interface will directly display the corresponding user information.

      • To add a member to your organization, follow these steps:

        • In the pop-up window, choose the organization role and the department that the member wants to join. Note that the role can be multi-selected. Please keep in mind that the member's access to the device in this department and sub-departments is determined by the role they have.

        • Once the role and department selection is complete, click 'Apply'. The system will then send an email to the user and wait for their confirmation.

    • Once a member is invited, they will appear in the list on the right side of the page with an "Unregistered" or "Unaccepted" status label next to their username. When the user clicks on the link in the email to complete the registration or authorization process, they will be added to the corresponding department and their label will be updated.

      However, if the user accidentally deletes the email or clicks "Reject" by mistake, they can still join by selecting their name from the list and clicking "Re-Invite". This will resend the invitation email to the user.

    6. Adding Devices¶

    To add a device under the Organization view, navigate to the Devices page where the department tree will be displayed on the left side of the page.

    For most business customers, devices will be added directly to the top departments, and there will be no need for any additional actions.

    1) Before adding a device, make sure to visit the router's admin panel. You need to Enable the Cloud Service.

    2) To add a device, follow the steps below: ① You need to select the target department. ② Click the "+ Add Device" button.

    There are three ways to bind devices to your GoodCloud account: Auto Discovery, Manual Add, and Bulk Import.

    1) Auto discover If your router and PC are on the same network, you can try the Auto Discovery feature. Follow the steps below to add a device: ① Click "Refresh" to enforce auto-discovery of devices. ② Enter the "Device ID" to verify that the router is original/valid. The "Device ID" can be found on the back of the router.

    2) Manually add If you are unable to use the Auto Discovery feature, you can manually add the device. Check the label on the back of the router for the device ID, MAC address, and S/N.

    3) Bulk import For those who need to add multiple devices at once, you can use the Bulk Import feature.

    7. Check Devices Information and Status¶

    Users can access the list of their devices on the left side under the 'Devices' section.

    • Batch Operations：

    • To view and edit device information and usage, click on the name of each device in the list. This will take you to the device details page. The middle tab bar on this page allows you to switch between different data of device usage, such as router overview, client list, timeline, sharing, and signal strength.

    • For IoT devices, you can view the following configuration information:

      • Bluetooth Configuration: Check the device's Bluetooth settings, including pairing status, Bluetooth version, connected devices, and Bluetooth signal strength.

      • Thread Configuration: View the device's Thread network settings, including network status, node role, connection status, and network name.

      • Please visit this website (https://docs.gl-inet.com/iot/en/thread_board_router/gl-s200/bluetooth/) for more information.

    8. Moving Devices to the Department¶

    Select the device on the right side of the page, and click the 'Manage' button to manage the device, including modifying device information, mobile grouping, mobile department, device sharing, and device deletion—five kinds of operations.

    To move the device to the desired department/group, select the department/group in the pop-up window and click on 'Confirm'. This operation ensures that different devices can be managed separately by different departments and enables the integration of the same type of devices. (For information on moving groups, refer to section 9 below - 'Adding Devices to a Group').

    Please note that members of each department can only view and operate devices under their own department without any special privilege assignment.

    9. Create a Device Group¶

    To create a device group, go to the Grouping page under the Organization View. The department tree is displayed on the left side of the page.

    • To create a group, select the department to which the group belongs from the list on the left and click on the 'Add' button.

    • In the pop-up window, enter the group name and check the device model and version. (You must add the device before you can create the group). Only one model and version of the device can be added to each group.

    • Once successfully added, the grouping will be displayed in the list on the right side of the page.

    10. Adding Devices to a Group¶

    To move devices to a device group, click on the left navigation bar to go to the Devices page. Select the devices you want to move to the device group and click on 'Manage' → 'Move Group'.

    Next, select the target group in the popup window and click on 'Confirm'.

    The number of devices in the corresponding group will increase. At this point, users can batch configure the devices in the device group, just like operating a single device.

    11. Creating a template for the device settings¶

    In the Organization view, click on the 'Settings Template' menu in the left navigation bar to access the page. The templates configured here can be used on the Devices and Groups pages.

    • The first step is to create a template. Click the 'Add' button to create a new template. In the pop-up window, set the template name, select the corresponding model and firmware version, and then click 'Next'.

    • In the 'Configuration' section on the left, select the relevant information to input, and then click 'Add'.

    • Preview the configuration details to be applied, and click 'OK' or 'Apply'.

    12. Apply the Template¶

    In the Organization view, switch to the Group/Device page using the left navigation bar. Locate the group or device you want to configure, and select it. Then, click on 'Settings' → 'Apply Template'.

    Next, in the pop-up window, select the template you have just configured and click 'Apply'.

    If you don't need to adjust the template, you can directly click the 'Apply' button to modify the device settings.

    To apply additional settings based on an existing template, click the 'Customize' button.

    13. Manage Devices¶

    Devices can be operated based on various customer requirements. Click on the 'Settings' operation button to modify the configuration and apply template operations.

    13.1 Modify the Configuration¶

    ① Select the configuration name you want to modify from the left menu bar.

    ② Input and select the specific parameters.

    ③ Click 'Apply'.

    ④ Review the new settings on the preview screen.

    ⑤ Click the 'Confirm' button to issue and apply the configuration.

    13.2 Share devices¶

    You can choose to share a device, granting the recipient full control over it. Please exercise caution when performing this action.

    By default, the device will be shared with an individual user, and you also have the option to set an expiration date for the sharing if needed.

    If the device experiences any issues, you can opt to share it with GL.iNet Technical Support along with some basic information. Our team will get in touch with you promptly to provide technical assistance.

    You can view and manage your device sharing settings from the device details page at any time.

    13.3 Upgrade devices¶

    ① Set the execution status and select the execution time.

    ② Click 'Next' to create the upgrade task.

    13.4 Device Control Operation¶

    Interact with the router's Admin management panel to achieve cloud control over the router.

    14. Adding a Device Alert Rule¶

    For Business users, effective management of both SIM card data usage and network connections is crucial for maintaining seamless operations and controlling costs.

    When a device's SIM card data usage approaches or exceeds a set threshold, or when the primary network link fails and switches to a backup, the system immediately notifies users.

    These real-time notifications help avoid unexpected overages, prevent service disruptions, and ensure business continuity.

    With enhanced monitoring across diverse business scenarios, the cloud platform empowers users to proactively manage device performance and maintain operational control.

    14.1 Managing Notification Rules¶

    Log into the GoodCloud platform. Navigate to the Notifications page from the main menu. Here, you can manage all alert rules, including adding, deleting, or modifying them.

    14.2 Adding a Notification Rule¶

    To create an alert rule, click the Add button on the Notifications page and follow these four steps:

    1) Select Monitoring Device.

    2) Set Trigger Event.

    3) Specify Recipients.

    4) Review Rule Settings.

    14.2.1 Select Monitoring Device¶

    Select Devices or Groups for which you want to create the rule.

    • To select specific devices, use the fuzzy search by Device Name；

    • The "Select All" option excludes devices from sub-departments; you can switch departments to select multiple monitoring targets.

    14.2.2 Set Trigger Event¶

    Set the Monitoring Criteria (e.g., SIM card data usage threshold, main link status).

    • You can switch event types to configure the monitoring parameters for the devices.

    14.2.3 Specify Recipients¶

    Choose the Recipients who will receive notifications (via email or internal message).

    • The "Select All" option for recipients excludes members from sub-departments; you can switch departments to select multiple recipients.

    14.2.4 Check Rule Settings¶

    You must first set the importance level for the alert rule. Once confirmed, apply the rule to activate monitoring.

    • The importance level set will apply to each trigger event within the rule, affecting notification methods and display on the page.

    14.2.5 Feature Highlights¶

    • The prompt displays the count of selected devices and device groups for your reference.

    • When monitoring alerts for a device group, any new device added will automatically sync existing alert rules for that group.

    • Certain alert rules can incorporate a debounce mechanism to prevent frequent alerts due to network or device issues, ensuring important messages aren't overlooked;

      • For example, with a 60-second Prevent Repeated Trigger mechanism, if a device goes offline and comes back online within 60 seconds, the alert rule won't trigger, and no internal messages or email notifications will be sent.

    • To modify previously set content, you have three options:

    - Click the steps bar at the top of the page；

- Click the edit button next to the preview content；

- Click the Previous button at the bottom right of the page；

    14.3 Editing / Deleting Notification Rules¶

    Step 1: On the Notifications page, view all existing notification rules. Use fuzzy search with Trigger Event, Device Name, Device Model, Description, and Recipients to quickly locate alert rules.

    Step 2: Click on any rule to edit or delete it.

    Step 3: Modify the devices, monitoring criteria, or recipients as needed, and save changes.

    14.4 Receiving and Viewing Notifications¶

    When a device triggers a notification rule, notifications will be sent via Email or Internal Message.

    14.4.1 Message Notification Reception¶

    • Email

    • Internal Messages

    • Message Box

    Alert information will be pinned in the message box, displaying only the five most recent unread messages. Click "View All" to see all unread messages.

    14.4.2 Accessing the Message Center for More Messages¶

    Internal messages can be viewed in the Message Center, where you can manage, acknowledge, or dismiss them.

    Quickly filter by message type, level, reception time, and read/unread status for historical tracking.

    14.4.3 Customizing Message Reception Settings¶

    Quickly filter by message type, level, reception time, and read/unread status for historical tracking.

    • Customize how users receive messages (internal messages/email) based on message categories, allowing selection of message types;

    • Each message type is divided into levels (aligned with alert rule settings), allowing different notification methods for the same message type.

    15. Site-to-Site¶

    In the Organization view, click the 'Site-to-Site' menu in the left navigation bar to access the page where you can create and manage network connections.

    For detailed instructions, please refer to the Site to Site.

    16. Brand Customization¶

    For White Label businesses, please refer to the WhiteLabel Domain for details on customizing your domain name. If you require assistance, please contact us directly.

    17. Organizational settings¶

    In the Organization view, click Settings in the left navigation bar to access the page where you can view security authentication and package information. Users and roles with permission to modify the organization's name can do so quickly from this page.

    18.API Integrations¶

    The Platform API is a RESTful interface that provides programmatic access to much of the data and actions in the system.

    To ensure the security of user data, users must create a token on the platform and manually submit it to the third-party integration system.

    To ensure the quality of interface data transmission and to provide clearer access to interface call records, we strongly recommend that users generate a token for each integration system they need to avoid token reuse.

    Please visit this website API Integrations for more information on API integration.
