    Functional Procedure¶

    1. Process Overview¶

    2.Join the VAR¶

    • Invitation and Registration:

      • When adding a user as a VAR administrator or member, an email invitation will be sent if the user has not yet registered for a GoodCloud account.

        • The user clicks the link in the email to register, and they will be automatically added to the VAR.

      • Sign Up:

        • Visit GoodCloud to sign up and sign in.

        • If you don't find the verification email, check your spam folder or try again later. For sign-up issues, email b2bsupport@gl-inet.com for assistance.

    • For Registered Users:

      • Method 1: An invitation email is sent to the user’s registered email address. Clicking the link will add them to the VAR and redirect them to the VAR’s view.

      • Method 2: When logged into GoodCloud, a reminder will pop up. Clicking "Accept" will add them to the VAR and redirect them to the VAR’s view.

    • View Status and Switching:

      • Users can view their current status in the upper right corner and switch between the VAR view, Organization view, and Self-view.

      • Note: Devices and data are not interoperable between individual accounts and VAR accounts.

    • Home Page Overview:

      • View metrics such as devices in stock, returned devices, total devices, online devices, offline devices, inactive devices, out-of-stock devices, and subordinate organizations.

      • Click on these metrics for quick access to detailed views.

    3. Creating an Organization in the VAR¶

    • Accessing the Organizations Page:

      • Click on 'Organizations' in the left navigation bar to view a list of organizations under this VAR, including details about the organization, its administrator, and the number of devices. Note that devices and data are not interoperable between different organizations under the VAR.

    • Adding a New Organization:

      • Click the '+ Add' button to start creating a new organization.

      • Enter the name of the organization in the dialog box and click 'Next'.

    • Setting Up an Administrator:

      • On the next page, set up an administrator for the organization. The administrator should be a customer of the VAR, not a VAR employee.

      • Enter the user’s email or username and specify the expiration date for the invitation. Click 'Next'.

    • Handling User Information:

      • If the email is not registered:

        • A prompt will ask if you want to invite the user to register. Click 'Confirm' and 'Apply' to send an invitation email. The user will register through the link in the email and join the organization upon successful registration.

      • If the email or username is already registered:

        • The system will display the existing user information. Simply click 'Apply'.

    • Completing the Creation:

      • Once the organization is successfully created, it will appear in the list. As the VAR owner, you can select the organization and click 'Access' to manage it.

    4. Creating and Modifying VAR & Organization Roles¶

    • Accessing Role Management:

      • In the left navigation bar, click on 'Role' Management to enter the page.

    4.1. Creating or modifying the VAR roles¶

    • Accessing the VAR Roles Page:

      • When you enter the Role Management page, it defaults to the VAR Role Add and Edit page.

    • Creating a New VAR Role:

      • Click the 'Add a New Role' button in the list.

      • Enter the role name at the top right of the page.

      • Configure the role by setting operational privileges for the functional modules listed on the right.

      • View Privileges: All functional modules must first have View privileges enabled before configuring other privileges.

      • Click "Allow All" or "Disable All" in the upper right corner to batch allow or disallow privileges.

      • Click 'Apply' to save and create the new role.

    • Modifying an Existing VAR Role:

      • Select a role you want to modify from the list.

      • Click 'Edit' in the lower right corner to enter the role editing mode.

      • Adjust the role name and permissions as needed.

      • Click 'Apply' to save changes.

    4.2. Creating or modifying the organization roles¶

    • Accessing the Organization Roles Page:

      • Click 'Organizational Roles' in the upper tab bar to manage roles specific to organizations.

      • Creating or Modifying Organization Roles:

      • Add and edit organization roles using the same method described in 3.1 for VAR roles.

      • Note: The organization roles created by the VAR are synchronized to the VAR's organization. Subordinate organizations cannot delete or edit these VAR-created roles.

    5. Inviting members to join the VAR¶

    Access the Members Page: Click on 'Members' in the left navigation bar. This page displays a list of members. VAR members are typically VAR employees responsible for managing transfers, configuring, and commissioning devices.

    Invite a Member: Click the 'Invite' button at the top of the list. The invitation process consists of two steps:

    • Step 1: Enter Invitation Details

      • Enter the email address or username of the user you want to invite in the pop-up window.

        • For Unregistered Emails: If the email is not registered, a pop-up will ask if you want to invite the user to register. Click 'Confirm' and 'Apply' to send an invitation email. The user must click the link in the email to register and be added to the VAR.

        • For Registered Emails: If the email is already registered, the system will display the user’s information.

      • Set the expiration date for the invitation (default is one hour; can range from one hour to 180 days).

      • Click 'Next'.

    • Step 2: Assign Roles and Permissions

      • Select the VAR role for this member in the next pop-up window. The role determines what the member can do with devices and data.

      • Choose the level of access:

        • 'None': No access to VAR subordinate organizations.

        • 'Partial Organizations': Allows configuration of specific organizations. Click '+ Add' to select the organizations the member can access and configure corresponding permissions.

          • After selecting the organizations, click 'Next' to return to the 'Member Settings' page.

          • Click 'Apply' to finalize the access settings.

        • 'All Organizations': If this option is selected, the invited employee will be granted access to all organizations under the VAR. Configure the unified permission roles and set a deadline for the employee’s access.

          • Please note that once this option is successfully configured, any new organization added by the VAR will automatically grant the employee the relevant rights based on the role permissions set at that time.

    • After configuring the access rights for this member, click the 'Apply' button to complete the invitation. This setup is typically used for VAR technical support staff, allowing them to access customer data and devices, and collaborate with customers to configure the devices.

    Other Actions: Invited members are listed on the right side of the page, with an "Unregistered" or "Unaccepted" status label next to their usernames.

    Once they complete the registration or authorization process by clicking the link in the email, their status will update accordingly.

    If a user accidentally deletes the email or clicks "Decline," you can select the member from the list and click "Re-Invite" to resend the invitation.

    To modify the VAR Roles for an invited member or adjust their roles related to VAR-affiliated organizations, click "Modify."

    6. Adding Devices¶

    To add devices directly to the customer's VAR account, follow these steps:

    • Access the Device Inventory Page:

      • Click 'Devices' in the left navigation bar.

      • Select 'Inventory' to enter the inventory page.

    • Add New Devices:

      • Click the '+ Add' button.

      • Enter the device information in the popup window.

      • Note: This process is the same as the one described in "For Organizational Customers - 6 Adding Devices."

    7. Classifying Devices in the Warehouse¶

    To classify devices in the warehouse:

    • Select Devices:

      • Choose the devices you want to classify.

    • Edit Device Information:

      • Click on the 'Edit' button.

    • Add Categorization Tags:

      • Enter categorized information (tags) via the keyboard.

      • Confirm the tags by pressing the Enter key.

    Note: Proper tagging facilitates easier device transfer and management.

    8.Transferring Devices¶

    • Filter Devices:

      • Use the search function in the upper right corner and advanced search options to find the desired devices.

        ① Fuzzy matching can be performed for device ID, MAC, SN, storage warehouse, and order number, and logical customized matching can also be achieved through advanced search;

        ② Select the matching conditions (all or any);

        ③ Add condition: A maximum of five items can be added;

        ④ The matching logic is divided into 3 parts, which are matching parameters, logical relationships, and search contents;

        ⑤ After clicking Apply, the matching result of the final device needs to satisfy all the matching logic.

      Searches are generally conducted in the following MAC filter form:

    • Transfer Devices:

      ①Select Devices: Choose the filtered devices.

      ② Click Transfer: Click the 'Transfer' button.

      ③ Select Organization: Choose the customer organization to receive the devices in the pop-up window.

      ④ Apply: Confirm the transfer.

    • View Outgoing Devices:

      • Switch to Organization View:

      • From the Home Page: Click on Online Devices, Offline Devices, Inactive Devices, or Total Devices.

      • Through Navigation: Use the 'Devices' menu on the left side of the navigation bar to jump to device pages.

        • Note: Organizations created under the VAR cannot add devices on their own; they can only receive devices transferred by the VAR.

    9. Returning Devices (Organization Member Operation)¶

    For a Deivce that has been shipped out to an organization and needs to be returned if there is a hardware failure, you can perform a return operation in the corresponding organization view of the VAR.

    • Access Organization View:

      • Enter the subordinate organization view.

      • Click the 'Devices' menu on the left navigation bar.

    • Initiate Return Process:

      • Select the devices to be returned under the department.

      • Click 'Manage' and then 'Return Devices' to start the return process.

    • Complete Return Details:

      • Fill in the reason for the return.

      • Confirm the basic information of the devices to be returned twice.

      • Click 'Confirm' to complete the return.

    • Return Confirmation:

      • The system will confirm that the return is successful.

      • Retain the return order number for future reference, which will be used by the VAR for inventory or deletion operations of the returned devices.

    10. Returned Devices Processing (VAR Member Operation)¶

    • Alert for Returned Devices:

      • The VAR will receive a notification when a device is returned.

    • View Returned Devices:

      • Go to Devices → Returning to view the list of returned devices.

    • Process Returned Devices:

      • Inventory:

        • If the device is not faulty and should be returned to the warehouse, select the device and click 'Inbound'.

      • Edit the returned device information if needed for future inventory management.

      • Delete:

        • If the device is damaged or defective and needs to be returned to GL.iNet or discarded, select the device and click 'Delete'.

    11. Creating Template settings¶

    The templates configured here can be used in the subordinate organization views 'Devices' and 'Groups' pages;

    • Access Template Configuration:

      • In the VAR view, click on the 'Template Configuration' menu in the left navigation bar.

    • Create a New Template:

      • Click the 'Add' button to start creating a new template.

      • In the pop-up window, set the template name, select the corresponding model and firmware version, and click Next.

    • Configure Template Information:

      • Enter the required configuration details and click Next.

    • Preview and Apply:

      • Review the configuration information.

      • Click Apply to finalize and create the template.

    • Manage Templates:

      • You can edit, clone, delete, and synchronize templates in the template list.

      • Edit: Modify existing templates.

      • Clone: Create a copy of a template.

      • Delete: Remove a template from the list.

      • Synchronize: Sync templates to the subordinate organization's template configuration list.

    12. Template Pre-Configuration¶

    • Access Brand Configuration Template:

      • In the VAR view, click the 'Modify' button to adjust the branding setup template. This template will be applied to uninitialized devices (including new or reset devices) when they come online in any organization.

    • Pre-Configuration Process:

      ① Click 'Modify': Access the brand configuration template settings.

      ② Enable Template Switch: Toggle the switch to enable the brand configuration template.

      ③ Output Configuration Information: Enter and review the branding information.

      ④ Confirm Configuration: Verify and confirm the configuration content.

    13. Apply the template¶

    • Navigate to Group/Device Page:

      • Enter the organization view under VAR.

      • Click the left navigation bar to switch to the Group/Device page.

    • Apply the Template:

      • Find and select the group you want to configure.

      • Click Settings → Apply Template.

      • In the pop-up window, select the template you configured and click Apply.

      • The log of the device settings will reflect the applied template.

    14. Brand customization¶

    Business background: For White Label business only. Custom domain name configuration details can be found in the WhiteLabel Domain. Contact us if you need further assistance.

    Steps:

    • Review the White Label Domain Guide for detailed instructions on customizing your domain name.

    • Contact GoodCloud support if you require assistance with configuring your White Label brand information or have specific needs related to the branding process.

    • Follow the provided guide to set up your domain name and other brand details as required for your White Label setup.

    15. VAR Information¶

    • View Distributor Information:

      • In the VAR view, click Distributor Information in the left navigation bar.

      • View security authentication and package information.

    • Modify Distributor Name:

      • Users with permission can quickly modify the distributor name.

    16. API Integrations¶

    The Platform API is a RESTful interface that provides programmatic access to much of the data and actions in the system.

    To ensure the security of user data, users must create a token on the platform and manually submit it to the third-party integration system.

    To ensure the quality of interface data transmission and to provide clearer access to interface call records, we strongly recommend that users generate a token for each integration system they need to avoid token reuse.

    Please visit this website API Integrations for more information on API integration.
